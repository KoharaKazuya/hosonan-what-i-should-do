# AGENTS.md

このリポジトリは Hosonan で利用するユーザーリポジトリです。AI エージェントは、このファイルを最初に読み、リポジトリ固有の設定を尊重してください。

## Hosonan の設定ファイル

### `hosonan.json`

`hosonan.json` は、Hosonan がこのリポジトリを扱うための機械可読な設定ファイルです。サイト名、公開 URL、ディレクトリ構成、タイムゾーン、表示や同期に関わる設定など、リポジトリ全体の動作に関わる情報を置きます。

AI エージェントは `hosonan.json` を次の方針で扱ってください。

- JSON として正しい構文を保つ
- 既存キーの意味を推測で変更しない
- リポジトリ全体の挙動に関わる変更だけを `hosonan.json` に入れる
- ユーザーの好み、関心領域、運用方針などは `hosonan.json` ではなく `config/` に入れる
- 破壊的なパス変更や公開 URL の変更は、ユーザーから明示された場合だけ行う

`hosonan.json` が存在しない場合は、必要になったときに既存の Hosonan の仕様に合わせて追加してください。

### `config/`

`config/` は、AI エージェント向けのユーザー設定ディレクトリです。特定のファイル名に依存せず、`config/` 以下にある設定ファイルを読み取って、ユーザーの好みや運用方針を把握してください。

`config/` には、たとえば次のような情報を書いてください。

- ユーザーの関心領域
- 優先して扱う企業、技術、市場、地域
- 好みの文体、表記、構成
- 避けたい話題や慎重に扱うべき観点
- リポジトリ運用上の方針

AI エージェントは `config/` を次の方針で扱ってください。

- 作業前に `config/` 以下のファイルを読み、設定内容を総合して解釈する
- 特定のファイル名が必ず存在する前提を置かない
- 設定同士が矛盾する場合は、より具体的で新しい内容を優先する
- ユーザーの好みや編集方針を追加・変更する場合は `config/` 以下に置く
- 機密情報、API キー、個人情報、未公開情報を `config/` に書かない

## GitHub Actions

workflow は `.github/workflows/` 以下に置きます。利用側リポジトリには、workflow が必要とする secrets や variables を GitHub 側で設定してください。

AI エージェントが workflow を変更する場合は、既存の secrets、variables、permissions、schedule、checkout、commit / push の挙動を不用意に壊さないでください。

## 変更時の確認

AI エージェントがこのリポジトリを変更した場合は、変更内容に応じて次を確認してください。

- JSON を変更した場合は、JSON として parse できること
- workflow を変更した場合は、YAML として読めること
- Markdown を変更した場合は、見出し構造とリンクが壊れていないこと
